01 — Pengenalan

Apa itu Self-Organizing Map?

SOM adalah jaringan saraf tiruan (neural network) yang menggunakan competitive learning untuk memetakan data berdimensi tinggi ke dalam peta 2D, sambil mempertahankan hubungan topologi antar data. Ditemukan oleh Teuvo Kohonen (1982), seorang ilmuwan komputer dari Finlandia.

Prinsip Utama

  • Data serupa dipetakan ke posisi berdekatan pada peta
  • Mengurangi dimensi sambil menjaga struktur topologi
  • Tidak membutuhkan label — unsupervised learning
  • Hasil berupa peta visual yang mudah diinterpretasi
02 — Arsitektur

Struktur Jaringan SOM

Komponen Utama

  • Input layer: menerima data berdimensi-n
  • Output layer: grid 2D berisi neuron-neuron
  • Setiap neuron memiliki vektor bobot berdimensi sama dengan input
  • Neuron terhubung dengan tetangganya di grid

Tipe Grid

  • Rectangular: 4 tetangga per neuron, lebih sederhana
  • Hexagonal: 6 tetangga per neuron, lebih merata dan umum digunakan
02 — Arsitektur

Best Matching Unit (BMU)

Ketika sebuah data input masuk, SOM menghitung jarak antara input dengan semua vektor bobot di grid. Neuron yang vektor bobotnya paling mirip (jarak terkecil) dengan input disebut Best Matching Unit (BMU).

BMU = argminj || x − wj ||

Neighborhood Function

Setelah BMU ditemukan, BMU dan tetangga-tetangganya di grid akan di-update. Neuron yang lebih dekat ke BMU di grid mendapat update lebih besar. Fungsi neighborhood yang umum digunakan adalah Gaussian.

03 — Algoritma

Langkah-Langkah Algoritma SOM

1

Inisialisasi: Tetapkan vektor bobot setiap neuron di grid secara acak atau menggunakan PCA dari data input.

2

Kompetisi: Pilih satu data input secara acak, lalu hitung jarak Euclidean ke semua neuron. Neuron dengan jarak terkecil menjadi BMU (pemenang).

3

Kooperasi: Tentukan tetangga BMU di grid menggunakan fungsi Gaussian. Neuron yang lebih dekat ke BMU mendapat pengaruh lebih besar.

4

Adaptasi: Update vektor bobot BMU dan tetangganya agar lebih mendekati input:   wj(t+1) = wj(t) + α(t) · h(j,BMU,t) · (x − wj(t))

5

Ulangi: Kurangi learning rate (α) dan radius neighborhood (σ) secara bertahap. Ulangi langkah 2-4 untuk ribuan iterasi hingga peta stabil.

03 — Algoritma

Rumus Matematis SOM

Weight Update

wj(t+1) = wj(t) + α(t) · h(j, BMU, t) · [x(t) − wj(t)]

Gaussian Neighborhood

h(j, BMU, t) = exp(−||rj − rBMU||² / 2σ(t)²)

Parameter Decay

α(t) = α₀ · exp(−t / τα)
σ(t) = σ₀ · exp(−t / τσ)

04 — Perhitungan Manual

Langkah 1: Cari BMU

Hitung jarak Euclidean dari input x = (3, 1) ke setiap neuron:

d(x, N1) = √[(3−0.45)² + (1−0.89)²]
= √[6.50 + 0.01] = 2.552
d(x, N2) = √[(3−0.55)² + (1−0.83)²]
= √[6.00 + 0.03] = 2.456
d(x, N3) = √[(3−0.95)² + (1−0.32)²]
= √[4.20 + 0.46] = 2.160 ← terkecil!
d(x, N4) = √[(3−0.62)² + (1−0.78)²]
= √[5.66 + 0.05] = 2.390
BMU: N3 (jarak 2.160)
04 — Perhitungan Manual

Langkah 2: Hitung Neighborhood & Update Bobot

Hitung nilai Gaussian neighborhood h(j, BMU) dengan σ = 0.6, lalu update setiap bobot:

Neighborhood Values

h(N3,N3) = exp(0) = 1.000 (BMU sendiri)
h(N1,N3) = exp(−1.0/0.72) = 0.249
h(N4,N3) = exp(−1.0/0.72) = 0.249
h(N2,N3) = exp(−2.0/0.72) = 0.062

Update BMU (N3)

w₃ = (0.95, 0.32) + 0.5 × 1.0 × (2.05, 0.68)
= (1.975, 0.66)

N3 bergerak paling banyak mendekati input. Neuron lain juga bergerak, tetapi lebih sedikit sesuai nilai h.

04 — Perhitungan Manual

Langkah 3: Update Semua Neuron

Setiap neuron di-update berdasarkan rumus wj(baru) = wj(lama) + α · h · (x − wj). BMU mendapat update terbesar, yang lain lebih kecil.

N1 (h = 0.249)

w₁ = (0.45, 0.89) + 0.5 × 0.249 × (2.55, 0.11)
= (0.45 + 0.317, 0.89 + 0.014) = (0.767, 0.904)

N4 (h = 0.249)

w₄ = (0.62, 0.78) + 0.5 × 0.249 × (2.38, 0.22)
= (0.62 + 0.296, 0.78 + 0.027) = (0.916, 0.807)

N2 (h = 0.062, paling jauh)

w₂ = (0.55, 0.83) + 0.5 × 0.062 × (2.45, 0.17)
= (0.55 + 0.076, 0.83 + 0.005) = (0.626, 0.835)

04 — Perhitungan Manual

Dua Fase Training SOM

Fase 1: Ordering (Self-Organizing)

  • ~1000 iterasi pertama
  • Learning rate besar (α ≈ 0.1–1.0)
  • Radius neighborhood besar (setengah grid)
  • Peta "membuka lipatan" dan menata diri secara topologi — pergerakan bobot drastis
  • Organisasi kasar terbentuk

Fase 2: Convergence (Fine-Tuning)

  • Rekomendasi: ≥ 500 × jumlah neuron iterasi
  • Learning rate kecil (α ≈ 0.01)
  • Radius kecil (1–2 neuron)
  • Peta menyesuaikan detil — pergerakan bobot halus
  • Akurasi statistik representasi dioptimalkan
04b — Dari SOM ke Cluster

SOM Saja Belum Menghasilkan Cluster!

Kesalahpahaman Umum

Banyak tutorial hanya menunjukkan training SOM (update bobot) tanpa menjelaskan bagaimana mendapatkan label cluster akhir. SOM sejatinya adalah dimensionality reduction + vector quantization, bukan clustering langsung.

Pipeline Lengkap (3 Tahap)

1

Train SOM — Update bobot iteratif hingga konvergen. Hasilnya: grid neuron dengan vektor bobot terlatih.

2

Cluster-kan neuron SOM — Terapkan K-Means, Hierarchical, atau analisis U-Matrix pada vektor bobot neuron untuk mengelompokkan neuron ke dalam cluster. (Tahap yang sering terlewat!)

3

Assign label ke data — Setiap data asli mendapat label cluster dari BMU-nya. Data → BMU → Cluster BMU = Cluster data.

04b — Dari SOM ke Cluster

Dua Pendekatan Mendapatkan Cluster dari SOM

Pendekatan A: Grid Kecil (1 neuron = 1 cluster)

Gunakan grid kecil dimana jumlah neuron = jumlah cluster yang diinginkan. Contoh: grid 1×3 untuk 3 cluster.

Grid 1×3 → 3 neuron → 3 cluster
  • Sederhana, langsung dapat label
  • Kehilangan manfaat preservasi topologi
  • Cocok untuk kasus sederhana
som = MiniSom(1, 3, n_features) # Setiap neuron = 1 cluster label = som.winner(x) # → (0,0) atau (0,1) atau (0,2)

Pendekatan B: Two-Stage Clustering (Standar)

Gunakan grid besar (misal 10×10), lalu cluster-kan neuron-neuron dengan K-Means atau Hierarchical Clustering pada vektor bobot.

Grid 10×10 → 100 neuron → K-Means → K cluster
  • Standar akademik (Vesanto & Alhoniemi, 2000)
  • Noise reduction: bobot = rata-rata lokal data
  • Efisien: cluster 100 neuron, bukan 100.000 data
# Tahap 2: Cluster vektor bobot weights = som.get_weights().reshape(-1, n) neuron_labels = KMeans(3).fit_predict(weights) # Tahap 3: Assign ke data asli data_label = neuron_labels[som.winner(x)]

05 — Konteks

Inspirasi Biologis: Peta Otak

SOM terinspirasi oleh cara korteks otak manusia mengorganisir informasi sensorik. Neuron-neuron yang merespons stimulus serupa terletak berdekatan secara fisik di otak.

Contoh di Otak

  • Somatotopic map: Jari tangan dipetakan berdekatan di korteks sensorik — jari telunjuk dan jari tengah merespons di area yang bersebelahan
  • Tonotopic map: Frekuensi suara rendah dan tinggi dipetakan berurutan di korteks pendengaran
  • Retinotopic map: Area yang berdekatan di retina dipetakan ke area berdekatan di korteks visual

SOM meniru prinsip ini secara komputasional — data yang mirip dipetakan ke neuron yang berdekatan di grid.

Panduan Pemilihan Parameter

ParameterRekomendasi
Grid size≈ 5 × √(jumlah data), misal 1000 data → grid 16×16
α awal0.1 – 1.0 (turun secara eksponensial)
σ awalSetengah diameter grid
IterasiFase 1: ~1000, Fase 2: ≥500 × jumlah neuron
Topologi gridHexagonal (6 tetangga) lebih baik dari rectangular (4 tetangga)
InisialisasiPCA lebih stabil, random lebih umum

Quality Measures

  • Quantization Error: Rata-rata jarak data ke BMU-nya. Semakin kecil semakin baik — menunjukkan seberapa akurat peta merepresentasikan data.
  • Topographic Error: Fraksi data yang BMU dan second-BMU-nya tidak bersebelahan di grid. Semakin kecil semakin baik — menunjukkan kualitas preservasi topologi.

07 — Perbandingan

Kelebihan & Kekurangan SOM

Kelebihan

  • Visualisasi data berdimensi tinggi ke dalam peta 2D yang intuitif
  • Mempertahankan hubungan topologi (data mirip tetap berdekatan)
  • Mampu menangkap hubungan non-linear antar fitur
  • Unsupervised — tidak butuh label data
  • Menghasilkan clustering sekaligus dimensionality reduction

Kekurangan

  • Komputasi mahal, terutama untuk dataset besar dan dimensi tinggi
  • Hasil non-deterministik (tergantung inisialisasi acak)
  • Ukuran grid harus ditentukan manual — sulit menentukan optimal
  • Tidak bisa menangani data kategorikal secara langsung
  • Tidak ada deteksi noise/outlier secara eksplisit
08 — Implementasi

Kode Python: Pipeline Lengkap SOM → Cluster

from minisom import MiniSom from sklearn.cluster import KMeans import numpy as np # ── Tahap 1: Train SOM ── som = MiniSom(10, 10, data.shape[1], sigma=1.5, learning_rate=0.5) som.random_weights_init(data) som.train_random(data, 5000) # ── Tahap 2: Cluster neuron ── weights = som.get_weights() codebook = weights.reshape(-1, data.shape[1]) km = KMeans(n_clusters=3) neuron_labels = km.fit_predict(codebook) # ── Tahap 3: Assign label ke data ── labels = [] for x in data: bmu = som.winner(x) # (row, col) idx = bmu[0] * 10 + bmu[1] labels.append(neuron_labels[idx]) print("Cluster labels:", labels)

Tahap 1: Train SOM

Grid 10×10 = 100 neuron. Training 5000 iterasi. Setiap neuron menyimpan vektor bobot.

Tahap 2: Cluster Neuron

get_weights() → reshape menjadi (100, n_fitur) → K-Means mengelompokkan 100 neuron ke 3 cluster. Ini tahap krusial!

Tahap 3: Assign Label

Setiap data → cari BMU → ambil label cluster BMU tersebut. Ini output akhir clustering!
