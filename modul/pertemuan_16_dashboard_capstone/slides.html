Dasar

API Dasar Streamlit + DuckDB

import streamlit as st import duckdb st.title("Dashboard Analitik") st.metric("Total Revenue", "Rp 12.5M", "+8%") # Koneksi DuckDB con = duckdb.connect('data.duckdb') df = con.sql("SELECT * FROM trips LIMIT 1000").df() st.dataframe(df) st.bar_chart(df.set_index('kota')['revenue'])

Streamlit = Python script biasa. Tidak perlu HTML/CSS/JS. Setiap st.xxx() langsung jadi komponen UI.

Layout Components

# Kolom sejajar col1, col2, col3 = st.columns(3) col1.metric("Trips", "1.2M") col2.metric("Revenue", "Rp 45B") col3.metric("Rating", "4.7") # Sidebar untuk filter with st.sidebar: kota = st.selectbox("Kota", ["Jakarta", "Bandung"]) # Tabs tab1, tab2 = st.tabs(["Overview", "Detail"]) # Expander & Container with st.expander("Lihat SQL"): st.code(query)

KPI

Pattern

KPI Cards Pattern

# KPI Row dengan delta (naik/turun) col1, col2, col3, col4 = st.columns(4) col1.metric( label="Total Trips", value="1,234,567", delta="+12.3%" # hijau (positif) ) col2.metric( label="Revenue", value="Rp 45.2B", delta="+8.1%" ) col3.metric( label="Avg Rating", value="4.72", delta="-0.03", delta_color="inverse" # merah (turun) ) col4.metric( label="Cancel Rate", value="3.2%", delta="-0.5%", delta_color="inverse" # hijau (turun=bagus) )

Performa

Caching: Jangan Query Ulang!

# Cache data — di-cache berdasarkan parameter @st.cache_data def load_data(kota): con = duckdb.connect('data.duckdb') return con.sql(f"SELECT * FROM trips WHERE kota='{kota}'").df() # Cache resource — untuk koneksi DB, model ML @st.cache_resource def get_connection(): return duckdb.connect('data.duckdb') # Tanpa cache: setiap interaksi = query ulang (lambat!) # Dengan cache: query sekali, simpan di memory

@st.cache_data untuk DataFrame/hasil query. @st.cache_resource untuk koneksi DB & model ML.

Fitur

SQL Explorer Component

# SQL Explorer — user bisa tulis query sendiri st.subheader("SQL Explorer") query = st.text_area( "Tulis SQL query:", value="SELECT kota, COUNT(*) as total\nFROM trips\nGROUP BY kota", height=150 ) if st.button("Run Query"): try: con = get_connection() result = con.sql(query).df() st.dataframe(result) st.download_button( "Download CSV", result.to_csv(), "result.csv" ) except Exception as e: st.error(f"Error: {e}")

Tips &
Kesalahan Umum

04

Tips

Tips Dashboard Profesional

  • Responsive Layout: Gunakan st.columns() agar tampilan rapi di berbagai layar. Test di mobile!
  • Caching Wajib: @st.cache_data untuk semua query. Tanpa cache, dashboard lambat setiap klik.
  • Error Handling: Bungkus query dengan try/except. Tampilkan st.error() yang informatif.
  • Loading State: Gunakan st.spinner("Loading...") saat proses berat berjalan.
  • Jangan Overload: Maksimal 4–6 chart per halaman. Gunakan tabs untuk organisasi.
  • Gunakan st.set_page_config(): Set layout="wide", page_title, page_icon di baris pertama.

Hindari

Kesalahan Umum Capstone

  • Terlalu ambisius: Pilih scope yang realistis. Lebih baik 3 komponen yang berfungsi sempurna daripada 7 setengah jadi.
  • Tidak ada architecture diagram: Tanpa diagram, penilai tidak mengerti flow data Anda.
  • Code tidak reproducible: Pastikan bisa di-run dari nol. Sertakan requirements.txt / pip install di cell pertama.
  • Dashboard tanpa insight: Jangan hanya tampilkan data mentah. Berikan interpretasi dan highlight temuan menarik.
  • Tidak latihan demo: Demo gagal = nilai presentasi turun drastis. Siapkan backup screenshot.
