Perbandingan

Pandas vs Polars

Pandas (2008)

  • Backend: NumPy (C)
  • Single-threaded
  • Eager execution SAJA
  • 1 GB data → 5-10 GB RAM
  • Index-based (df.iloc, df.loc)
  • Mutable (inplace=True)

Polars (2021)

  • Backend: Rust + Apache Arrow
  • Multi-threaded otomatis
  • Lazy + Eager evaluation
  • 1 GB data → ~1.5 GB RAM
  • Expression API (pl.col())
  • Immutable (lebih aman)

Polars bukan "Pandas yang lebih cepat" — ini paradigma BERBEDA.

◊

Fondasi

Apache Arrow Memory Format

Standar memori kolom yang dipakai Polars, DuckDB, Spark, dan 40+ project lain. Data di memori TIDAK perlu di-copy saat berpindah antar library.

Zero-Copy

Polars ↔ DuckDB ↔ Spark tanpa copy data. Hemat RAM & waktu.

Columnar

Data disimpan per kolom, bukan per baris. Ideal untuk analytics.

Strongly Typed

Int64, Float64, String, Date — tidak ada "object" seperti Pandas.

Interoperable

Polars → DuckDB → Spark → R: semua pakai format yang SAMA.

Konsep Kunci

Eager vs Lazy Evaluation

Eager (Pandas/default)

  • filter → scan seluruh data (1x)
  • group_by → scan seluruh data (2x)
  • sort → scan seluruh data (3x)
  • Total: 3x full scan!
  • Setiap operasi LANGSUNG dijalankan

Lazy (Polars LazyFrame)

  • filter → catat saja
  • group_by → catat saja
  • sort → catat saja
  • .collect() → OPTIMASI + 1x scan!
  • Query optimizer menggabungkan operasi

Predicate pushdown: filter dipindah ke awal. Projection pushdown: hanya baca kolom yang dibutuhkan.

API

Expression API — Cara Polars Bekerja

Polars TIDAK pakai index. Semua operasi menggunakan Expression via pl.col().

# Pandas style (index-based) df[df['kota'] == 'Jakarta']['tarif'].mean() # Polars style (expression-based) df.filter(pl.col('kota') == 'Jakarta').select(pl.col('tarif').mean()) # Tambah kolom baru df.with_columns( (pl.col('tarif') * 1.11).alias('tarif_ppn'), pl.col('rating').round(0).alias('rating_bulat') ) # Group by + Aggregation df.group_by('kota').agg( pl.col('tarif').mean().alias('avg_tarif'), pl.col('tarif').sum().alias('total_rev'), pl.len().alias('jumlah') )

Performa

scan_parquet vs read_parquet

read_parquet (Eager)

  • Baca SELURUH file ke memori
  • 10M rows = 800 MB RAM langsung
  • Baru kemudian filter/select
  • Seperti baca seluruh buku

scan_parquet (Lazy)

  • Hanya baca METADATA dulu
  • Filter → baca baris yang lolos SAJA
  • Select → baca kolom yang dipilih SAJA
  • Seperti baca daftar isi, lalu halaman tertentu
# Lazy: hanya baca kolom kota & tarif, hanya baris Jakarta result = ( pl.scan_parquet('trips_10m.parquet') .filter(pl.col('kota') == 'Jakarta') .select('kota', 'tarif') .collect() )
RFM

Studi Kasus

RFM: Segmentasi Pelanggan

Teknik NYATA yang dipakai Tokopedia, Shopee, GoJek untuk mengelompokkan pelanggan.

R
Recency
Kapan terakhir beli? Semakin baru = semakin baik. Selisih hari dari transaksi terakhir.
F
Frequency
Seberapa sering beli? Pelanggan loyal beli berkali-kali dalam sebulan.
M
Monetary
Berapa total belanja? High-value customer belanja lebih banyak.

Score 1-5 per metrik (qcut). R=5,F=5,M=5 → VIP. R=1,F=1,M=1 → At Risk / Churned.

Segmentasi

RFM Score → Segmen Pelanggan

SegmenRFMAksi
VIP / Champions555Loyalty reward, early access
Loyal Customer3-54-53-5Upsell, referral program
Potential Loyalist4-52-32-3Engage lebih, membership
New Customer511-2Welcome series, onboarding
At Risk1-23-53-5Win-back campaign, diskon
Hibernating111Reactivation atau let go

Insight bisnis: 20% pelanggan VIP biasanya menghasilkan 80% revenue (Pareto principle).

Konsep

Cohort Analysis: Pantau Retensi

Kelompokkan pelanggan berdasarkan bulan pertama kali beli (cohort), lalu track berapa % yang masih aktif di bulan-bulan berikutnya.

Cohort

Kelompok pelanggan berdasarkan bulan pertama transaksi (Jan, Feb, Mar...)

Retention Rate

% pelanggan yang kembali membeli di bulan ke-1, ke-2, ke-3...

Cohort Period

Selisih bulan antara bulan pertama dan bulan transaksi berikutnya

Heatmap

Visualisasi retention — warna gelap = retensi tinggi, terang = rendah

GoJek benchmark: retensi bulan ke-1 = 40-50%, bulan ke-6 = 15-20% dianggap BAGUS.

Lazy Evaluation — Demo

# Lazy: bangun query plan, belum dieksekusi lazy_q = ( pl.scan_parquet('trips_10m.parquet') .filter(pl.col('kota') == 'Jakarta') .filter(pl.col('tarif') > 50000) .group_by('layanan') .agg( pl.col('tarif').mean().alias('avg_tarif'), pl.len().alias('jumlah') ) .sort('avg_tarif', descending=True) ) # Lihat query plan (SEBELUM eksekusi) print(lazy_q.explain()) # Eksekusi! Semua operasi digabung + dioptimasi result = lazy_q.collect() print(result)

.explain() menunjukkan: FILTER dipush ke bawah, hanya kolom kota/layanan/tarif yang dibaca!

Praktikum 3

RFM Implementation dengan Polars

import polars as pl from datetime import date # Hitung RFM per customer (id sebagai customer proxy) snapshot = date(2025, 1, 1) rfm = df.group_by('id').agg( (pl.lit(snapshot) - pl.col('tanggal').max()).dt.total_days().alias('recency'), pl.len().alias('frequency'), pl.col('tarif').sum().alias('monetary'), ) # Score 1-5 dengan qcut rfm = rfm.with_columns( pl.col('recency').qcut(5, labels=['5','4','3','2','1'], allow_duplicates=True).alias('R'), pl.col('frequency').qcut(5, labels=['1','2','3','4','5'], allow_duplicates=True).alias('F'), pl.col('monetary').qcut(5, labels=['1','2','3','4','5'], allow_duplicates=True).alias('M'), ) print(rfm.head(10))

Praktikum 3 (lanjutan)

RFM → Segmen Pelanggan

# Gabungkan score RFM rfm = rfm.with_columns( (pl.col('R').cast(pl.String) + pl.col('F').cast(pl.String) + pl.col('M').cast(pl.String)).alias('rfm_score') ) # Klasifikasi segmen rfm = rfm.with_columns( pl.when( (pl.col('R').cast(pl.Int32) >= 4) & (pl.col('F').cast(pl.Int32) >= 4) & (pl.col('M').cast(pl.Int32) >= 4) ).then(pl.lit('VIP')) .when( (pl.col('R').cast(pl.Int32) <= 2) & (pl.col('F').cast(pl.Int32) >= 3) ).then(pl.lit('At Risk')) .when( (pl.col('R').cast(pl.Int32) >= 4) & (pl.col('F').cast(pl.Int32) <= 2) ).then(pl.lit('New Customer')) .otherwise(pl.lit('Regular')) .alias('segmen') ) # Hitung distribusi segmen rfm.group_by('segmen').agg(pl.len().alias('count')).sort('count', descending=True)

Praktikum 4

Cohort Analysis dengan Polars

# Step 1: Tentukan cohort (bulan pertama transaksi per customer) cohort = df.with_columns( pl.col('tanggal').dt.truncate('1mo').alias('order_month') ).with_columns( pl.col('order_month').min().over('id').alias('cohort_month') ) # Step 2: Hitung cohort period (bulan ke-N) cohort = cohort.with_columns( ((pl.col('order_month') - pl.col('cohort_month')) .dt.total_days() / 30).cast(pl.Int32).alias('period') ) # Step 3: Hitung retention cohort_table = ( cohort.group_by('cohort_month', 'period') .agg(pl.col('id').n_unique().alias('customers')) .sort('cohort_month', 'period') )

04

Hindari

Kesalahan Umum Polars

  • pl.count() sudah DEPRECATED → gunakan pl.len()
  • qcut tanpa allow_duplicates=True → CRASH jika ada nilai duplikat. Selalu tambahkan parameter ini.
  • pl.Utf8 sudah DEPRECATED → gunakan pl.String
  • read_parquet untuk data besar → boros RAM. Gunakan scan_parquet (lazy).
  • Lupa .collect() pada LazyFrame → tidak ada hasil! LazyFrame hanya rencana, belum eksekusi.
  • Pakai .apply() atau loop → sangat lambat. Gunakan Expression API (pl.col(), pl.when()).

Ekosistem

Polars dalam Ekosistem Data Modern

Polars + DuckDB

Query Polars DataFrame pakai SQL: duckdb.sql("SELECT * FROM pldf")

Polars + Matplotlib

pl_df.to_pandas() → plt.plot(). Atau langsung .to_series().to_list()

Polars + scikit-learn

pl_df.to_numpy() → model.fit(). Polars untuk preprocessing, sklearn untuk ML.

Polars + Parquet

Polars jadi "glue" antara data prep dan tools lain — berkat Apache Arrow zero-copy.

Ringkasan

Apa yang Sudah Kita Pelajari

  • Polars = DataFrame library berbasis Rust, 5-50x lebih cepat dari Pandas
  • Apache Arrow = format memori kolom standar industri, zero-copy antar library
  • Lazy evaluation = catat operasi, optimasi, eksekusi sekali. Predicate & projection pushdown.

  • RFM Segmentation = Recency + Frequency + Monetary → VIP, At Risk, New Customer
