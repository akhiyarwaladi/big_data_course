DBSCAN

Density-Based Spatial Clustering
of Applications with Noise

01 — Pengenalan

Apa itu DBSCAN?

DBSCAN adalah algoritma clustering yang mengelompokkan titik-titik data berdasarkan kepadatan (density) area di sekitarnya. Berbeda dari K-Means yang menggunakan jarak ke centroid, DBSCAN menemukan cluster berdasarkan daerah yang padat. Diperkenalkan oleh Martin Ester et al. pada konferensi KDD tahun 1996.

Prinsip Utama

  • Daerah padat membentuk cluster
  • Daerah sepi menjadi pemisah
  • Titik terisolasi = noise
  • Bentuk cluster bisa bebas
02 — Konsep Dasar

Dua Parameter Utama

ε Epsilon (eps)

Bayangkan setiap titik memiliki 'jarak pandang' berbentuk lingkaran. Semua titik lain yang berada dalam lingkaran ini dianggap sebagai tetangga. Semakin besar epsilon, semakin banyak tetangga yang ditemukan.

MinPts

Jumlah minimum titik yang harus ada dalam radius epsilon (termasuk titik itu sendiri) agar dianggap sebagai daerah yang padat. Seperti 'quorum' dalam rapat — berapa orang minimal agar bisa disebut kerumunan?

|Nε(p)| ≥ MinPts → Core Point
02 — Konsep Dasar

Tiga Tipe Titik

Core Point

Memiliki ≥ MinPts tetangga dalam radius ε. Menjadi inti pembentuk cluster.

Border Point

Dalam radius ε dari core point, tetapi < MinPts tetangga sendiri.

Noise Point

Tidak dalam radius ε dari core manapun. Dianggap outlier.

03 — Algoritma

Cara Kerja DBSCAN

1

Pilih titik acak yang belum dikunjungi dari dataset sebagai titik awal pemeriksaan.

2

Hitung semua titik tetangga dalam radius ε. Jika jumlah tetangga ≥ MinPts, maka titik ini adalah core point — buat cluster baru dan masukkan titik ini.

3

Ekspansi cluster: Periksa semua tetangga dari core point. Tambahkan ke cluster. Jika tetangga juga merupakan core point, lanjutkan ekspansi secara rekursif dari tetangga tersebut.

4

Jika jumlah tetangga kurang dari MinPts dan titik tidak berada dalam jangkauan core point lain, tandai titik tersebut sebagai noise (outlier).

5

Ulangi langkah 1-4 untuk semua titik yang belum dikunjungi hingga seluruh dataset selesai diproses.

04 — Iterasi 1

Kunjungi A → CORE → Buat Cluster 1

Proses

  • A belum dikunjungi → pilih A
  • Cek tetangga radius ε=1.5:
  • N(A) = {A, B, C, H} → 4 titik
  • 4 ≥ MinPts(3) → A = CORE POINT
  • Buat Cluster 1, masukkan A
  • Masukkan tetangga ke antrian
Cluster 1: {A}Antrian: [B, C, H]
04 — Iterasi 2

Ambil B dari Antrian → CORE → Ekspansi

Proses

  • Ambil B dari antrian [B, C, H]
  • N(B) = {B, A, C, H} → 4 titik
  • 4 ≥ 3 → B = CORE POINT
  • Cek tetangga B untuk ekspansi:
  • A → sudah di Cluster 1
  • C → sudah di antrian
  • H → sudah di antrian
  • Tidak ada titik baru
Cluster 1: {A, B}Antrian: [C, H]
04 — Iterasi 3

Ambil C dari Antrian → CORE → Ekspansi

Proses

  • Ambil C dari antrian [C, H]
  • N(C) = {C, A, B, H} → 4 titik
  • 4 ≥ 3 → C = CORE POINT
  • A, B → sudah di Cluster 1
  • H → sudah di antrian
  • Tidak ada titik baru
Cluster 1: {A, B, C}Antrian: [H]
04 — Iterasi 4

Ambil H → CORE → Cluster 1 Selesai!

Proses

  • Ambil H dari antrian [H]
  • N(H) = {H, A, B, C} → 4 titik
  • 4 ≥ 3 → H = CORE POINT
  • A, B, C → semua sudah di Cluster 1
  • Antrian kosong
  • ✓ Cluster 1 selesai!
Cluster 1: {A, B, C, H} ✓Antrian: [ ] Selesai!
04 — Iterasi 5

Kunjungi D → CORE → Buat Cluster 2

Proses

  • D belum dikunjungi → pilih D
  • N(D) = {D, E, F} → 3 titik
  • 3 ≥ 3 → D = CORE POINT
  • Buat Cluster 2, masukkan D
  • Masukkan E, F ke antrian
Cluster 2: {D}Antrian: [E, F]
04 — Iterasi 6 & 7

E → CORE, F → CORE → Cluster 2 Selesai!

Iterasi 6 — Ambil E

  • Ambil E dari antrian. Cek tetangga: N(E) = {E, D, F} — ada 3 titik
  • 3 ≥ MinPts(3), sehingga E juga merupakan core point
  • D sudah di Cluster 2, F sudah di antrian — tidak ada titik baru untuk ditambahkan

Iterasi 7 — Ambil F

  • Ambil F dari antrian. Cek tetangga: N(F) = {F, D, E} — ada 3 titik
  • 3 ≥ MinPts(3), F juga core point. D dan E sudah di cluster
  • Antrian sekarang kosong — Cluster 2 selesai dengan anggota {D, E, F}!
Cluster 2: {D, E, F} ✓Sisa: G belum
04 — Iterasi 8 (Terakhir)

Kunjungi G → NOISE → Algoritma Selesai!

Proses

  • G adalah satu-satunya titik yang belum dikunjungi dalam dataset
  • Cek tetangga: N(G) = {G} — hanya ada 1 titik (dirinya sendiri)
  • 1 < MinPts(3), sehingga G bukan core point
  • G juga tidak berada dalam radius epsilon dari core point manapun yang sudah ditemukan
  • Kesimpulan: G diklasifikasikan sebagai NOISE dengan label = -1

06 — Perbandingan

Kelebihan & Kekurangan

Kelebihan

  • Tidak perlu menentukan K di awal
  • Cluster berbentuk arbitrer (non-convex)
  • Robust terhadap outlier
  • Efisien dengan spatial indexing
  • Deterministik untuk core points

Kekurangan

  • Sensitif terhadap ε dan MinPts
  • Sulit untuk kepadatan berbeda
  • Performa menurun di dimensi tinggi
  • Border points non-deterministik
  • Perlu metrik jarak bermakna
07 — Parameter Tuning

Memilih ε dan MinPts

MinPts — Rule of Thumb

  • MinPts ≥ D + 1 (D = dimensi)
  • Data 2D → minimal MinPts = 4
  • Aturan umum: MinPts = 2 × dimensi

Tips Praktis

  • Selalu normalkan data dulu
  • Gunakan Silhouette Score untuk evaluasi
  • Grid search beberapa kombinasi

ε — k-Distance Graph

Plot jarak ke tetangga ke-k yang diurutkan. Cari "elbow" → ε optimal.

09 — Implementasi

Contoh Kode Python

from sklearn.cluster import DBSCAN import numpy as np X = np.array([ [1,2], [2,2], [2,3], [8,7], [8,8], [9,7], [5,5], [1,3] ]) db = DBSCAN(eps=1.5, min_samples=3) labels = db.fit_predict(X) print(labels) # Output: [0 0 0 1 1 1 -1 0]

Penjelasan

  • Label 0 → Cluster 1 (A, B, C, H) — keempat titik berdekatan di kiri bawah
  • Label 1 → Cluster 2 (D, E, F) — tiga titik berdekatan di kanan atas
  • Label -1 → Noise (G) — titik terisolasi di tengah, tidak punya tetangga cukup
